# ADR-001 Spec 文档治理规则

- 状态：accepted
- 当前性：当前有效
- 日期：2026-04-20

## 背景

项目在多轮迭代后出现了“后续 feature 改了旧行为，但旧正文没回写”的问题，读者从头阅读时无法判断当前版本。

## 决策

采用“决策 / 架构 / 模块 / feature / 变更”分层文档体系，并执行以下规则：

1. 同一主题只能有一份当前主口径
2. 正文只表达当前行为，历史不混入主叙述
3. 后续迭代若改变旧行为，必须同步回写旧文档或标记 `superseded`
4. 总索引负责回答“现在应该先看哪份”
5. 结构重组时，先更新模块文档和 change note，再处理 feature 细节

## 影响

- `specs/README.md` 与 `specs/spec.md` 成为阅读主入口
- `changes/` 成为口径切换和结构映射的固定容器
- 旧编号文档可保留，但必须通过状态字段说明当前性

#### 功能描述
本需求描述两类本地存储对象：

1. 图片级“标记对象”：服务搜索增强，包括手动标记和自动提示词
2. 私有角色卡片：服务冷门角色或私有对象召回，不属于图片标签体系本身

两者职责不同，不再混用同一套“通用角色知识”或“知识库标签”语义。

#### 标记对象定义
图片级标记继续使用结构化对象存储，作为搜索辅路信号：

```ts
TagObject {
  text: string,
  category: "meme" | "person" | "source" | "custom",
  isAuto: boolean,
  sourceStrategy: "manual" | "ocr" | "file_name" | "ocr+file_name" | "clip_text" | "example_image" | "fallback",
  confidence?: number
}
```

说明：
- `manual` 表示用户主动维护的高价值标记
- 其他来源统一视为自动提示词来源
- `confidence` 允许保留，但不再直接参与主排序

#### 标记存储规则
- 每张图片的标记以结构化记录保存
- 手动标记与自动提示词可并存
- 用户修改自动提示词后，保存结果按用户最终输入覆盖原自动属性
- 文本相同但分类不同的标记默认视为冲突，不重复存储
- 手动标记默认可落在 `custom`

#### 标记分类语义
| 分类 | 标识值 | 说明 | 示例 |
|-----|-------|------|------|
| 梗 | meme | 用于表达梗点、语气或常见反应 | 破防了、绷不住了 |
| 人物 | person | 用于表达人物、角色、主体 | 老板、阿布 |
| 出处 | source | 用于表达影视、节目、作品来源 | 甄嬛传、让子弹飞 |
| 自定义 | custom | 用户主动维护的私域标记 | 工作群、阴阳、常用 |

#### 私有角色卡片定义
私有角色卡片用于支持“无字图 + 冷门角色 + 模型不认识”的检索场景，对外推荐 schema 为：

```ts
PrivateRoleCard {
  name: string,
  category: "person",
  aliases: string[],
  match_terms?: string[],
  notes?: string,
  match_mode?: "exact" | "contains" | "exact_or_contains",
  priority?: number,
  example_images: string[]
}
```

说明：
- `name` 是角色主名称
- `aliases` 用于昵称、别称、圈内叫法
- `match_terms` 用于动作、表情、场景等记忆点
- `notes` 仅供维护人理解，不参与核心匹配
- `example_images` 是私有角色主路的必要条件
- 维护工具默认按 `category = person` 创建角色卡片，不再要求维护人手动配置分类
- `match_mode / priority` 允许在兼容层保留，但不再作为维护工具默认暴露字段

#### 兼容规则
为兼容历史文件，运行时允许读取旧字段：

| 旧字段 | 新字段 | 兼容策略 |
|-------|-------|---------|
| `canonical` | `name` | 读取时映射，写回时优先写 `name` |
| `description` | `notes` | 读取时映射，写回时优先写 `notes` |

说明：
- 新增或编辑角色卡片时，不应继续把 `canonical / description` 作为首选字段暴露给维护人
- 旧文件可无迁移直接读取

#### 可用私有角色卡片判定
运行时只有满足以下条件的角色卡片才会进入私有角色主路：
- `category = person`
- `name` 非空
- `example_images` 至少有 1 张

若缺少示例图：
- 角色卡片允许保存
- 但运行时只保留弱文本匹配，不触发私有角色主路

#### 标记与角色卡片的关系
| 对象 | 作用 | 是否附着在图片上 | 是否参与主路 |
|-----|------|----------------|-------------|
| 手动标记 | 私域快速检索增强 | 是 | 否 |
| 自动提示词 | OCR/文件名弱辅助 | 是 | 否 |
| 私有角色卡片 | 冷门角色主体召回 | 否 | 是 |

说明：
- 标记对象属于“图片元数据”
- 私有角色卡片属于“角色库资产”
- 两者可在搜索时协同，但不应在模型层混为同一对象

#### 校验规则
**标记对象：**
- `text` 不能为空
- 文本建议最长 `50` 字符
- 同图同文本标记需去重

**私有角色卡片：**
- `name` 必须唯一
- `aliases` 与 `match_terms` 内部应去重
- `category` 必须为 `person`
- `example_images` 可为空，但为空时应给出降级提示
- 不允许高歧义超短词直接作为强角色别名

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| 手动标记与自动提示词同文 | 同图上同文本重复 | 自动去重，保留手动标记 | 无 |
| 私有角色卡片缺示例图 | `example_images` 为空 | 允许保存，不启用私有角色主路 | "缺少示例图，将降级为普通搜索" |
| 角色卡片分类错误 | `category != person` | 允许保存，但不视为可用私有角色卡片 | "当前条目不是私有角色卡片" |
| 读取旧 schema | 文件中只有 `canonical / description` | 自动映射为 `name / notes` | 无 |

说明：
- 上述两类提示属于运行时约束提示，不应再退化成“建议补充示例图”之类的泛化校验文案

#### 验收标准
**功能指标：**
- [ ] 图片标记对象支持结构化存储
- [ ] 私有角色卡片与图片标记对象在模型层明确区分
- [ ] 私有角色卡片支持 `name / aliases / example_images / notes`
- [ ] 旧字段 `canonical / description` 可兼容读取

**约束指标：**
- [ ] 标记对象不承担私有角色主路职责
- [ ] 私有角色卡片不回写为图片标签
- [ ] 缺少示例图的角色卡片不会误触发私有角色主路

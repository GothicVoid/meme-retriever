#### 功能描述
图片入库时，系统基于 OCR 文本、可读文件名以及后续可扩展的弱视觉线索，生成少量自动提示词，作为后续搜索的辅助信号。自动提示词不是最终真值标签，不要求完整覆盖图片内容，也不影响图片是否能成功入库。

#### 目标说明
- 为未维护文件名、未手动打标的图片补充少量检索线索
- 优先提取用户真正可能记得住的短词，而不是追求结构化完备
- 控制自动结果数量，避免噪声提示词污染搜索排序
- 保证自动提示词失败或为空时，图片仍可正常入库和被 OCR / 语义搜索找到

#### 输入信号
| 信号来源 | 优先级 | 作用 |
|---------|-------|------|
| OCR 文本 | 高 | 提取图片上可见的高价值短语，是首期主来源 |
| 文件名 | 低 | 仅在文件名可读时提供弱辅助，不依赖用户主动维护 |
| 示例图匹配 | 预留 | 后续可用于补充私有角色或视觉主体线索，本期不作为核心前提 |

#### 录入流程
```text
图片入库
  → 执行 OCR
  → 清洗 OCR 结果
  → 提取高价值短语
  → 若文件名可读，补充少量文件名提示词
  → 去重、截断、限制数量
  → 作为自动提示词写入 tags 表
```

#### 提取原则
- 优先保留用户会在搜索时直接输入的短语，如图片上的字、称呼、口头禅
- 不要求自动提示词形成完整分类体系
- 不要求自动提示词覆盖图片全部主体、动作、场景信息
- 单张图片自动提示词数量应严格受限，避免“多写多得分”
- 若 OCR 文本质量差或无文本，允许自动提示词为空

#### OCR 提示词规则
- 优先提取长度适中的高价值文本片段
- 对明显无意义的噪声文本、过长整句、乱码进行过滤
- 若 OCR 中存在多个候选，优先保留区分度更高的短词
- OCR 结果仍应保留在 OCR 索引中；自动提示词只是额外摘要，不替代 OCR 原文

#### 文件名提示词规则
- 仅当文件名本身可读、包含自然语言片段时才参与
- 系统生成的默认文件名、哈希名、时间戳名不应产出提示词
- 文件名提示词权重低于 OCR 提示词
- 文件名只作为补充来源，不应单独主导搜索排序

#### 数量与去重约束
- 单张图片自动提示词总数建议不超过 `5`
- 同一文本去重后仅保留一条
- 多来源命中同一文本时合并为一条记录
- 截断时优先保留 OCR 提示词，其次保留可读文件名提示词

#### 失败与降级策略
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| OCR 为空 | 图片无文本或识别失败 | 不生成 OCR 提示词，继续入库 | 无 |
| 文件名不可读 | 哈希名、纯数字名、随机串 | 不生成文件名提示词 | 无 |
| 提示词全部被过滤 | 结果全部为噪声或重复 | 自动提示词留空，继续入库 | 无 |
| 自动流程异常 | 清洗或生成过程报错 | 跳过自动提示词写入，不阻塞入库 | "自动提示词生成失败，已跳过" |

#### 与搜索阶段的关系
- 自动提示词仅作为搜索辅路信号
- 自动提示词可帮助补充 OCR 未完全覆盖的记忆点
- 自动提示词不能压过明显的 OCR 主路或高质量语义主路
- 用户后续添加的手动标记优先级高于自动提示词

#### 验收标准
**功能指标：**
- [ ] 入库时可生成少量自动提示词
- [ ] 自动提示词可来自 OCR 和可读文件名
- [ ] 自动提示词为空时不影响图片入库
- [ ] 自动提示词结果会参与后续搜索辅助排序

**约束指标：**
- [ ] 自动提示词不被当作最终真值标签
- [ ] 文件名提示词仅作为弱辅助来源
- [ ] 单张图片自动提示词数量受控

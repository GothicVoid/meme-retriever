#### 功能描述
在已完成首页启动态与搜索体验增强的基础上，对产品整体信息架构做一轮 P2 重构。目标不是增加更多页面，而是让用户能清楚理解“去哪里找图”“去哪里整理图库”“去哪里顺手调整少量配置”“哪些能力属于高级维护能力”。

本次重构重点解决以下问题：
- 当前页面入口更接近内部功能清单，而不是用户任务路径
- 图库页更像陈列页，未承担“搜索失败后去修”的管理职责
- 私有角色库作为高级能力过于显眼，容易让普通用户误以为必须理解这套概念
- 旧设置模块容易继续膨胀，暴露超出普通用户理解范围的底层策略

#### 设计目标
- 用户进入应用后，能在 1 次导航判断内理解每个页面负责什么
- 搜索失败后，用户知道应前往图库管理页处理图片缺失、补充信息或查看新增内容
- 普通用户默认不会被“私有角色库”“开发调试”等高级概念打断
- 不再保留旧设置模块，避免继续演变成内部控制面板

#### 范围说明
**本期做什么：**
- 重构顶层导航的信息层级与默认文案
- 明确首页/搜索、图库管理两类用户主任务页
- 将私有角色库降级为高级能力，移出普通用户主导航
- 将图库页从陈列页调整为管理页，补充管理型入口与状态分组
- 收缩设置页范围，并将开发者/维护人能力做二级分层

**本期不做什么：**
- 不新增复杂工作台、多级后台或可自定义导航系统
- 不改变搜索引擎底层召回与排序算法
- 不将私有角色库开放为普通用户默认必经流程
- 不引入权限登录、账号体系或在线协作

#### 页面架构总览
重构后的一级页面应收敛为以下三类：

| 一级入口 | 页面职责 | 主要用户问题 | 是否默认可见 |
|---------|---------|-------------|-------------|
| 首页 / 搜索 | 启动搜索、继续上次使用、查看搜索结果 | “我想马上找到图” | 是 |
| 图库管理 | 导入、批量整理、查看新增、处理异常、补充信息 | “为什么搜不到，我要去修图库” | 是 |

补充规则：
- 私有角色库不再作为一级导航独立展示
- 开发调试能力不与普通用户入口并列展示
- 顶层导航文案优先使用任务语言，不使用工程术语

#### 页面职责拆分
**1. 首页 / 搜索**

首页 / 搜索页继续承担以下职责：
- 空查询时作为首页启动态
- 非空查询时作为搜索结果页
- 提供搜索历史、最近使用、常用内容和失败反馈闭环

首页 / 搜索页不应承担以下职责：
- 不作为图片批量整理页
- 不在首屏暴露私有角色维护入口
- 不让用户在搜索主路径中接触开发调试概念

**2. 图库管理**

图库管理页应从“图片陈列列表”升级为“图库管理工作页”，承担以下职责：
- 导入图片或追加图片
- 批量选择、批量删除、批量整理
- 查看最近新增图片
- 查看异常图片，如原文件丢失、索引失败、信息缺失
- 对单图执行补充信息动作，如编辑标签、检查 OCR、进入详情

图库管理页不应承担以下职责：
- 不替代搜索页做按记忆线索找图
- 不暴露底层检索权重和复杂策略控制

**3. 私有角色库**

私有角色库定位调整为高级增强能力：
- 默认不在普通用户主导航中展示
- 仅在图库管理的高级能力区、开发模式或受控入口中展示
- 文案上明确其适用于冷门角色、私有对象和高阶维护场景

#### 导航规则
**推荐一级导航顺序：**
1. 首页 / 搜索
2. 图库管理

**导航展示规则：**
- 展开态下，应使用稳定可见的一级导航，明确承接 `首页 / 搜索`、`图库管理` 两类主入口
- 侧边栏态下，可弱化一级导航占位，由搜索页底部的更多菜单承接跨页动作和少量帮助入口，避免压缩主搜索空间
- 当前页应具有明显选中态
- 一级导航数量控制在 2 项，避免再次膨胀
- 私有角色库入口若存在，只能放在图库管理的高级能力区或受控入口中，不得与首页 / 搜索、图库管理并列
- 开发调试相关入口仅在开发模式下显示，且位置晚于普通设置项

#### 用户流程说明
**主流程 1：用户进入应用并找图**
1. 用户进入应用
2. 系统默认进入首页 / 搜索页
3. 用户通过首页启动态或搜索框发起搜索
4. 若找到目标图，用户直接复制或查看详情
5. 若搜索失败，页面提供“去图库管理看看”的明确引导

**主流程 2：搜索失败后去图库修复**
1. 用户在首页 / 搜索页搜索失败或结果不理想
2. 用户点击“去图库管理”入口
3. 图库管理页默认打开可执行的管理视图，如最近新增、异常图片或当前筛选结果
4. 用户执行补充信息、删除异常图片或检查新增内容
5. 用户返回首页 / 搜索页再次检索

**主流程 3：高级用户维护私有角色库**
1. 高级用户进入图库管理
2. 在“高级能力”区域看到私有角色库入口及简短说明
3. 用户进入私有角色库维护工具
4. 完成角色卡片维护后返回图库管理或搜索页

#### 交互说明
**1. 一级导航**

页面元素：
- 一级导航栏
- 当前页标题
- 当前页说明文案或副标题

交互行为：
- 点击一级导航后，切换到对应页面
- 一级导航应保持页面标题与实际职责一致，例如 `图库管理` 不再使用模糊文案
- 如果首页与搜索共用同一路由，可在导航上统一显示为 `首页 / 搜索` 或 `搜索`
- 当应用处于展开态时，顶部工具栏直接展示 `首页 / 搜索`、`图库管理` 两个一级入口，并高亮当前页
- 当应用处于侧边栏态时，搜索页可将 `图库管理` 和少量帮助入口收纳到更多菜单中；该菜单属于当前页操作入口，不单独形成新的一级页面

反馈提示：
- 页面切换后，标题与首屏模块应让用户立即确认自己来到了正确的位置

**2. 图库管理页**

页面元素：
- 页面标题：`图库管理`
- 主操作按钮：`添加图片`
- 次操作按钮：`添加文件夹`
- 管理视图切换入口，至少包括 `全部图片`、`最近新增`、`异常图片`
- 批量操作区
- 图片列表或网格
- 单图快捷操作入口，如编辑标签、查看详情、删除

交互行为：
- 用户进入图库管理页时，应优先看到“我可以在这里管理什么”的结构化入口，而不是纯图片瀑布流
- 导入能力保留在图库管理页，但不再要求用户首次导入必须先进入该页
- `最近新增` 用于查看刚导入或最近入库的图片
- `异常图片` 用于处理原文件丢失、索引失败或信息异常的图片
- 单图项应允许快速进入详情或补充信息，不要求用户先从搜索结果里绕回
- 批量操作仅在有选中项时激活

空状态：
- 图库为空时，图库管理页应展示导入引导和功能说明，而不是仅显示空列表
- 某个分组为空时，应明确说明“当前没有异常图片”或“最近暂无新增”

**3. 图库管理中的高级能力**

页面元素：
- 高级能力区
- 私有角色库入口

交互行为：
- 普通用户默认不被该能力打断，但在需要时可在图库管理中找到
- 高级能力区应使用低干扰方式呈现私有角色库，例如说明文案 + 次级按钮

文案要求：
- 私有角色库入口文案应强调“适合维护少量冷门角色或私有对象”
- 禁止使用让普通用户误以为必须先配置角色库才能搜索的文案

#### 页面间跳转规则
| 来源页面 | 触发动作 | 目标页面 | 目标状态 |
|---------|---------|---------|---------|
| 首页 / 搜索 | 点击一级导航“图库管理” | 图库管理 | 默认视图 |
| 首页 / 搜索 | 搜索失败后点击“去图库管理” | 图库管理 | 建议优先进入与问题相关的视图，如异常图片或最近新增 |
| 图库管理 | 点击一级导航“首页 / 搜索” | 首页 / 搜索 | 保留最近一次搜索框内容或回到首页启动态 |
| 图库管理 | 点击私有角色库入口 | 私有角色库维护工具 | 进入高级能力维护工具 |
| 私有角色库维护工具 | 关闭或返回 | 图库管理 | 返回高级能力区 |

补充说明：
- 搜索失败引导到图库管理时，允许通过查询参数或本地状态指定默认视图，但不要求本期设计复杂深链协议
- 若当前实现中首页与搜索共用同一页面，返回时应尽量保留用户的最近上下文

#### 数据与状态要求
| 数据项 | 用途 | 最低要求 |
|-------|------|---------|
| `navigation_state` | 记录当前一级页面 | 支撑导航高亮与页面切换 |
| `gallery_view_mode` | 图库管理页当前视图 | 至少支持 `all` / `recent` / `issues` |
| `issue_image_count` | 异常图片数量 | 用于异常图片视图提示与空状态 |
| `recent_added_count` | 最近新增图片数量 | 用于最近新增视图提示 |
| `developer_mode` | 控制开发调试入口可见性 | 与现有设置项共用 |

补充要求：
- 图库管理页所需数据应独立于搜索结果数据源获取，不应直接复用搜索结果态冒充管理视图
- 私有角色库入口可见性应受受控入口规则或用户模式控制，避免普通用户误触

#### 权限说明
- 可访问角色：普通用户、高级用户、维护人
- 权限校验规则：
  - 普通用户可访问首页 / 搜索、图库管理、基础设置
  - 高级用户和维护人可在受控入口中访问私有角色库
  - 开发调试区仅在开发模式开启时显示
- 无权限时的处理：
  - 对普通用户隐藏私有角色库默认入口，而不是展示后再报错
  - 对未开启开发模式的用户隐藏开发调试入口

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| 图库为空 | 用户首次进入图库管理页且无任何图片 | 展示导入引导和页面说明 | `先导入图片，再在这里整理和排查问题` |
| 搜索失败后无可处理异常 | 用户从失败反馈进入图库管理，但异常图片为空 | 默认回退到最近新增或全部图片视图 | `当前没有异常图片，可先看看最近新增` |
| 普通用户未启用高级模式 | 用户进入图库管理 | 不展示私有角色库默认入口 | 无 |
| 开发模式关闭 | 普通使用场景 | 隐藏开发调试区 | 无 |
| 图库管理某分组为空 | 最近新增或异常图片为空 | 展示该分组空态，不影响其他分组使用 | `最近暂无新增` / `当前没有异常图片` |
| 私有角色库入口不可用 | 当前构建未包含维护工具或入口受限 | 图库管理展示禁用态说明或不展示入口 | `当前版本未启用私有角色维护工具` |

#### 迭代计划
- 后续可在图库管理页补充更细的异常类型筛选，如文件丢失、索引失败、待补信息
- 后续可在搜索失败时直接把失败查询带入图库管理页，辅助用户定位相关图片
- 后续可根据真实使用情况决定是否将 `首页 / 搜索` 文案进一步收敛为单一 `搜索`
- 后续可为高级用户提供更明确的“启用高级能力”开关，但不应影响普通用户主路径

#### 验收标准
**功能指标：**
- [ ] 展开态一级导航只保留 `首页 / 搜索`、`图库管理` 两类主入口
- [ ] 私有角色库不再作为普通用户一级导航独立展示
- [ ] 图库页具备 `全部图片`、`最近新增`、`异常图片` 等管理视图入口
- [ ] 搜索失败时，页面提供进入图库管理的明确路径
- [ ] 旧设置模块不再出现在主信息架构中

**体验指标：**
- [ ] 用户能理解每个一级页面负责什么，不会把图库管理误认为另一个搜索页
- [ ] 搜索失败后，用户知道可以去图库管理页处理图片问题或补充信息
- [ ] 普通用户在默认路径中不会被私有角色库等高级概念打断
- [ ] 用户不会再感知到一个独立“设置模块”存在

**可靠性指标：**
- [ ] 当图库为空、异常分组为空或高级入口受限时，页面仍能稳定展示对应空态
- [ ] 导航切换不会导致页面标题、入口文案和页面职责互相矛盾
- [ ] 开发模式与普通模式下的入口可见性规则保持一致

#### 假设
- 假设当前应用仍采用桌面端单窗口结构，一级导航切换不涉及多窗口管理
- 假设图库管理页可在现有图库列表页基础上演进，而不是完全重写新页面
- 假设私有角色库仍保留现有实现能力，本次主要调整其入口层级和对外语义

#### 开放问题
- 搜索失败跳转到图库管理时，默认应进入 `异常图片` 还是 `最近新增`，当前建议优先按失败类型决定，否则回退到 `最近新增`

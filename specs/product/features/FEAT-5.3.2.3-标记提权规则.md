#### 功能描述
搜索阶段保留 OCR 与图像语义作为主路，标记分支只提供增强信号。其中，用户主动维护的手动标记提供强提权；系统自动生成的提示词仅提供弱辅助。标记分支不再依赖分类权重和 confidence 主排序。

#### 设计原则
- 搜索主路仍由 OCR / 图像语义决定，标记只做增强
- 手动标记强于自动提示词，但不能完全压制 OCR 主路
- 自动提示词是弱辅助，不应因为数量多而获得不成比例优势
- 分类字段保留用于存储与展示，但本期不参与搜索主排序
- `confidence` 可保留为生成质量元数据，但本期不参与搜索主排序

#### 标签分支计分公式
标记分支建议按以下结构计算：

```
标记命中得分 = 文本匹配得分 × 来源系数
```

其中：
- `文本匹配得分`：描述查询与标记文本的匹配强度
- `来源系数`：仅区分手动标记与自动提示词

#### 文本匹配得分建议
| 命中方式 | 建议得分 | 说明 |
|---------|---------|------|
| 完全命中 | 1.0 | 查询与标记文本一致，或归一后等价 |
| 部分命中 | 0.8 | 查询与标记文本包含关系成立 |
| 关联命中 | 0.5 | 查询经相关词或兼容扩展后间接命中 |

#### 来源系数建议
| 来源类型 | 建议系数 | 说明 |
|---------|---------|------|
| 手动标记 | 1.0 | 用户主动维护，可信度最高 |
| 自动提示词 | 0.6 | 仅作为弱辅助，不主导搜索排序 |

说明：
- 本期不再细分 `ocr`、`file_name`、`ocr+file_name`、`clip_text`、`example_image` 的搜索权重差异
- 这些来源值仍可继续保留在存储层，用于调试、展示或后续扩展
- 自动提示词即使完全命中，仍应显著弱于 OCR 主路命中

#### 单图标记聚合规则
为避免标记多的图片天然占优，单图标记分支不做简单累加。

本期建议：
```
S_mark(image) = max(标记命中得分)
```

说明：
- 单张图片存在多个命中标记时，只取最高分
- 不因标记数量多而线性刷分
- 若同一图片同时存在手动标记和自动提示词，按最高命中项计分

#### 与现有搜索总公式的关系
总分仍沿用现有搜索融合框架，标记分支只作为辅路：
```
主路得分 = OCR 或 语义主路得分
辅路得分 = 轻量 OCR / 语义补充 + 标记增强
最终排序 = 主路得分 + 辅路得分 + 轻量热度修正
```

说明：
- 标记分支不是独立主路
- 手动标记可帮助“老板”“同事”“工作群”等私域图片排得更靠前
- 自动提示词只能做弱辅助，不能压过明显 OCR 命中结果

#### 搜索阶段命中优先级
搜索阶段建议按以下优先级判断标记命中：
1. 手动标记完全命中
2. 手动标记部分命中
3. 自动提示词完全命中
4. 自动提示词部分命中
5. 扩展相关词命中

#### 限制条件
- 标记分支不得恒定压过 OCR 主路和语义主路
- 对空查询不应用标记提权逻辑
- 自动提示词不得仅凭数量优势压过高质量 OCR 命中
- 本期不允许分类字段直接决定搜索排序高低
- 本期不允许 `confidence` 直接参与主排序公式

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| 同图多标记命中 | 多个标记同时命中查询 | 默认取最高分，避免叠加失真 | 无 |
| 自动提示词误命中 | 自动提示词与查询偶然重合 | 仅给弱辅助分，不主导结果 | 无 |
| 历史数据无分类 | 历史标记未写 category | 保留兼容，不影响本期排序 | 无 |
| 历史数据无 confidence | 历史标记未写 confidence | 保留兼容，不影响本期排序 | 无 |
| 来源值未知 | 历史标记缺失 sourceStrategy | 按自动提示词保守处理 | 无 |

#### 验收标准
**功能指标：**
- [ ] 手动标记优先级高于自动提示词
- [ ] 自动提示词只提供弱辅助，不主导搜索排序
- [ ] 搜索结果排序可解释，不出现明显失真
- [ ] 兼容历史分类字段和 confidence 字段
- [ ] 单图标记数量不会直接带来线性刷分

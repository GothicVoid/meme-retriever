### 5.3.2.1 自动标签录入流程（优先级：P1）

#### 功能描述
用户添加图片到图库后，系统在原有索引构建流程中自动执行标签分析，生成候选标签并保存为结构化标签记录。自动打标是入库的附加能力，失败时不应阻塞图片入库成功。

#### 流程说明
1. 用户添加图片或文件夹
2. 系统完成基础校验、去重和图片元数据写入
3. 执行 OCR 文本提取
4. 过滤并解析可读文件名，作为可选弱辅助信号
5. 基于知识库生成候选标签集合
6. 对候选标签执行文本匹配打分
7. 对重复候选进行归并、裁剪和阈值过滤
8. 写入自动标签记录
9. 更新搜索索引并返回进度结果

#### 推荐算法流程
```
输入：
- OCR 文本
- 过滤后的文件名
- 知识库条目

处理：
- 候选生成
- 文本匹配打分
- 来源合并
- 去重归并
- 阈值过滤
- Top N 截断

输出：
- 结构化标签列表
```

#### 打分策略
第一阶段建议采用以下信号源：
- `OCR 文本匹配`：OCR 文本命中知识库关键词、别名、规则片段
- `文件名弱匹配`：仅当文件名满足可读文本条件时，参与低权重补充匹配

后续增强信号：
- `CLIP 文本相似度`：用于无字图或人物图等特定场景的候选补充
- `示例图匹配`：作为扩展能力预留，不作为第一阶段上线阻塞项

定义说明：
- `CLIP 文本相似度`：将入库图片与知识库标签文本进行向量相似度比较，属于“图片检索标签文本”
- `示例图匹配`：将入库图片与知识库中某标签的示例图片进行向量相似度比较，属于“图片检索标签样本图”

职责差异：
- `CLIP 文本相似度` 更适合作为抽象视觉语义补充，如情绪、反应、未配置示例图的主体概念
- `示例图匹配` 更适合作为具体视觉对象识别补充，如人物、角色、影视截图来源

建议公式：
```
候选得分 = OCR文本匹配 * 1.0 + 文件名弱匹配 * 0.3
```

说明：
- 文件名仅用于补充，不得单独高置信入库
- 不满足可读文本条件的文件名直接忽略
- 分类权重不在入库阶段固化，只用于候选重排和搜索阶段
- 第一阶段不依赖 CLIP 自动产出标签，避免将搜索信号直接等价为标签真值
- 后续若启用示例图，优先将其作为候选补充或重排信号，而非唯一直接落库依据
- 后续若同时启用示例图与 CLIP 文本相似度，建议先示例图、后文本相似度，避免一次性引入过多语义噪声

#### 后续增强启用顺序
建议按以下顺序逐步启用：
1. `OCR 文本匹配`
2. `可读文件名弱匹配`
3. `示例图匹配`
4. `CLIP 文本相似度`

启用原则：
- 第一阶段先验证知识库文本规则与结构化标签链路
- 若无字图、人物图、出处图漏标明显，再补示例图
- 若示例图维护成本高或标签覆盖不全，再引入 CLIP 文本相似度做低置信补充

#### 分类与信号建议
| 标签分类 | 第一阶段主信号 | 第二阶段增强 | 第三阶段增强 |
|---------|---------------|-------------|-------------|
| meme | OCR 文本匹配 | 可选少量示例图增强 | CLIP 文本相似度补充 |
| person | 无或极弱 | 示例图匹配 | CLIP 文本相似度补充 |
| source | OCR 文本匹配 | 示例图匹配 | CLIP 文本相似度补充 |
| custom | 不自动生成 | 不适用 | 不适用 |

#### 文件名可读性规则
文件名仅在满足以下条件之一时参与弱匹配：
- 包含至少 2 个连续中文字符
- 包含明确可读词组，且不属于时间戳、随机串、哈希串或系统默认命名

以下类型默认忽略：
- `IMG_20240101_123456`
- `image`
- `微信图片_20240401`
- 长随机串、十六进制串、纯数字串

#### 交互与反馈
- 入库中显示“正在分析标签...”
- 单张图片自动打标失败时，图片仍可入库成功
- 若自动标签为空，不弹出错误，只记录为“无自动标签”
- 批量导入时不对每张图片单独打断用户操作

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| OCR 失败 | 图片无文字或 OCR 模型不可用 | 跳过 OCR 信号，仅尝试可读文件名弱匹配 | "未识别到图片文字，可手动补充标签" |
| 文件名不可读 | 文件名为乱码、编号、时间戳或随机串 | 忽略文件名，不参与候选生成 | 无 |
| OCR 与文件名都无有效信号 | 模型或输入无法提供文本线索 | 不写自动标签，保留空标签结果 | "未生成自动标签，可手动补充" |
| CLIP 不参与第一阶段 | 第一阶段方案约束 | 仅用于搜索，不参与自动标签生成 | 无 |
| 示例图未配置 | 知识库条目无示例图 | 不影响第一阶段，后续增强阶段跳过该信号 | 无 |
| 示例图与文本信号冲突 | 不同信号给出不同候选 | 以高优先级信号为准，低优先级仅作补充分 | 无 |
| 知识库无候选 | 无任何条目命中 | 不写兜底垃圾标签 | "未匹配到知识库标签，可手动补充" |
| 候选过多 | 单图候选数超过上限 | 仅保留高分 Top N | 无 |
| 单个候选重复 | OCR 与文件名命中同一标签 | 合并为一个标签并累积来源信息 | 无 |

#### 验收标准
**功能指标：**
- [ ] 新增图片时自动触发标签分析
- [ ] 自动打标失败不影响图片入库
- [ ] OCR 与文件名弱匹配可独立降级
- [ ] 单图自动标签数量可控，不产生明显噪声

**性能指标：**
- [ ] 自动打标附加耗时控制在索引总耗时预算内
- [ ] 候选筛选与入库写入不会成为主要瓶颈

### 5.5.2 知识库维护工具（优先级：P2）

#### 修订记录
| 版本号 | 修订日期 | 修订人 | 修订内容 | 修订原因 |
|-------|---------|--------|---------|---------|
| v1.0 | 2026-04-14 | 开发 | 新增知识库维护工具需求文档 | 支撑知识库录入、校验与调试，降低手写 JSON 成本 |

#### 功能描述
提供一套面向项目维护人的知识库维护工具，用于创建、编辑、校验和测试 `knowledge_base.json`。该工具属于内部维护能力，不面向普通用户暴露，不纳入正式产品主流程。

#### 目标说明
- 降低维护人直接手写知识库 JSON 的成本
- 提高知识库条目录入、修改和冲突校验效率
- 支持在当前项目规则下快速验证某条知识库是否能被 OCR 文本正确命中
- 保持知识库格式与主项目实现同步演进，避免外部工具与项目规则脱节

#### 工具形态
第一阶段建议实现为“同仓库内的内部维护工具”，不拆分为独立仓库或独立产品。

推荐形态优先级：
1. CLI 工具
2. 仅开发模式可见的维护页面
3. 独立外部工具

第一阶段结论：
- 工具与当前项目放在同一仓库
- 默认采用 CLI 形式
- 如后续确有需要，可增加开发模式专用页面
- 不作为普通用户可见功能发布

#### 使用角色
| 角色 | 描述 | 权限 |
|-----|------|------|
| 项目维护人 | 负责知识库录入、修订和规则治理的内部人员 | 可创建、编辑、校验、测试和导出知识库 |
| 普通用户 | 应用使用者 | 无知识库维护权限 |

#### 流程说明
1. 维护人启动知识库维护工具
2. 选择创建条目、编辑条目、校验知识库或测试匹配
3. 工具根据当前知识库 schema 执行输入校验
4. 工具输出修改结果或校验报告
5. 维护人确认后写回本地知识库文件
6. 维护人可进一步执行匹配测试，验证 OCR 文本能否命中目标标签

#### 核心功能
**基础能力：**
- 创建知识库条目
- 编辑已有条目
- 删除条目
- 查询和列出条目
- 按 `canonical`、`category`、关键词检索条目

**校验能力：**
- 校验 JSON 结构是否合法
- 校验 `canonical` 是否唯一
- 校验 `category` 是否合法
- 校验 `aliases` 与 `match_terms` 是否重复
- 检测潜在冲突词
- 检测高歧义短词

**测试能力：**
- 输入一段 OCR 文本，查看命中的知识库条目
- 显示命中类型、命中词和候选分值
- 预览最终会落库的 `canonical` 标签
- 在修改条目后快速回归测试

**输出能力：**
- 写回 `knowledge_base.json`
- 导出格式化后的知识库文件
- 输出校验报告或冲突报告

#### 推荐命令能力
若采用 CLI 形式，建议支持以下命令：
- `kb list`
- `kb add`
- `kb edit`
- `kb delete`
- `kb validate`
- `kb test-match --text "<OCR文本>"`
- `kb format`

#### 交互说明
**第一阶段交互原则：**
- 优先保证可用性和校验完整性
- 不追求复杂图形化界面
- 所有操作应有明确反馈
- 高风险操作需二次确认

**反馈示例：**
- 新增成功："已创建知识库条目：蚌埠住了"
- 校验失败："发现重复 canonical：甄嬛传"
- 冲突提示："词条‘皇上’同时命中多个条目，请人工处理"
- 匹配测试结果："输入文本命中 2 个候选，最终推荐标签：甄嬛传"

#### 数据规则
工具应遵循主项目当前知识库 schema，不允许定义独立于主项目之外的格式。

第一阶段至少支持以下字段：
- `canonical`
- `category`
- `aliases`
- `match_terms`
- `description`
- `match_mode`
- `priority`

工具不应绕过主项目定义的知识库校验规则。

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| canonical 重复 | 新增或编辑后与已有条目冲突 | 阻止保存 | "canonical 已存在" |
| category 非法 | 输入不在允许枚举中 | 阻止保存 | "非法分类" |
| aliases 重复 | 同条目内存在重复别名 | 自动去重或阻止保存 | "检测到重复 aliases" |
| match_terms 冲突 | 词条可命中多个条目 | 输出冲突报告，允许维护人继续或取消 | "检测到潜在冲突词" |
| 高歧义短词 | 新增单字或高歧义短词 | 给出警告，默认不建议保存 | "高歧义短词，请确认是否保留" |
| 文件格式损坏 | 知识库 JSON 不合法 | 阻止写回 | "知识库文件格式错误" |
| 测试无命中 | 输入文本未命中任何条目 | 返回空结果 | "未命中任何知识库条目" |

#### 产品边界
**本期做什么：**
- 面向维护人的知识库录入与校验工具
- 与当前项目规则强绑定
- 本地使用，不依赖联网

**本期不做什么：**
- 面向普通用户开放的知识库管理页面
- 独立发布的外部知识库产品
- 在线协作编辑
- 云端同步词库

#### 验收标准
**功能指标：**
- [ ] 维护人可通过工具完成知识库条目增删改查
- [ ] 可对知识库执行结构校验和冲突检测
- [ ] 可通过输入 OCR 文本测试命中结果
- [ ] 可安全写回格式化后的知识库文件

**体验指标：**
- [ ] 常见维护操作无需直接手写 JSON
- [ ] 校验错误信息清晰可定位
- [ ] 工具输出与主项目知识库规则保持一致

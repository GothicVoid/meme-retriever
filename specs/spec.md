# 1. 产品概述

## 1.1 背景与问题
在日常聊天场景中，用户往往知道自己想发送哪张表情包，却难以在本地图库中快速定位。传统的文件管理方式只能通过文件名或时间查找，当图库规模达到数百上千张时，"翻图"成本极高，严重影响沟通体验。

## 1.2 产品定义
一款基于语义检索的本地表情包管理工具，让用户通过自然语言描述即可秒找目标图片。

## 1.3 目标用户
| 用户类型 | 用户特征 | 核心诉求 | 使用场景 |
|---------|---------|---------|---------|
| 重度表情包用户 | 本地存有500+张表情包，日均使用10次以上 | 快速定位、快速复制、批量管理 | 日常聊天、社群互动 |
| 内容创作者 | 需要引用特定梗图制作内容，对图片出处有要求 | 按出处/角色/台词精准检索 | 短视频制作、文章配图 |
| 轻度用户 | 图库100张以内，偶尔使用 | 界面简洁、操作简单 | 偶尔发送表情包 |

## 1.4 产品目标
**业务目标：**
- 获得广泛的用户群体和积极的市场反馈
- 用户日均使用频次达到较高水平
- 通过社区口碑和用户推荐实现自然增长

**用户目标：**
- 降低用户查找表情包的时间成本
- 提升表情包管理效率

**成功指标：**
- 检索响应时间 < 500ms（图库≤5000张）
- 用户找到目标图片的平均操作次数 ≤ 2次
- 用户留存率（7日）达到较高水平

## 1.5 范围说明
### 本期做什么
- 本地语义检索功能
- 图库基础管理（添加、删除、查看）
- 标签系统
- 快捷复制功能
- 基础设置项

### 本期不做什么
- 本地图片文件管理
- 云端同步功能
- 表情包社区/分享功能
- 在线表情包下载
- 移动端适配
- 图片编辑功能
- 表情包制作工具

# 2. 系统概要

## 2.1 系统组成
系统由以下核心模块组成：

1. **检索引擎模块**：负责语义理解与图片匹配
2. **图库管理模块**：负责图片的增删改查
3. **索引构建模块**：负责图片内容分析与索引生成
4. **标签系统模块**：负责用户自定义标签管理
5. **设置模块**：负责应用配置管理
6. **UI交互模块**：负责用户界面展示与交互

## 2.2 数据流图
```
用户输入查询
    ↓
检索引擎模块 ← 读取 → 索引数据库
    ↓
计算相关性与热度得分
    ↓
返回结果列表 → UI展示
    ↓
用户操作（复制）
    ↓
更新使用频次 → 索引数据库

添加图片流程：
用户选择图片 → 图库管理模块 → 索引构建模块
                                    ↓
                            分析图片内容（OCR）
                                    ↓
                            生成索引 → 索引数据库
```

# 4. 数据字典与术语表

## 4.1 核心业务对象
| 对象名称 | 定义 | 关键属性 | 生命周期说明 |
|---------|-----|---------|-------------|
| 表情包图片 | 用户添加到图库的图片文件 | 图片ID、文件路径、添加时间、使用次数、标签列表 | 从用户添加到图库开始，直到用户删除为止 |
| 标签 | 用户为图片添加的文本标注 | 标签ID、标签名称、关联图片数 | 创建后持久存在，直到所有关联图片被删除 |
| 索引记录 | 图片的语义特征向量及元数据 | 图片ID、特征向量、OCR文本、识别内容 | 与图片生命周期一致，图片删除时同步删除 |

## 4.2 数据字典
| 字段名 | 字段说明 | 数据类型 | 取值范围/规则 | 是否必填 |
|-------|---------|---------|--------------|---------|
| image_id | 图片唯一标识 | String(UUID) | 系统自动生成 | 是 |
| file_path | 图片文件完整路径 | String | 有效的文件系统路径 | 是 |
| file_name | 原始文件名 | String | 最长255字符 | 是 |
| file_format | 图片格式 | Enum | JPG/PNG/GIF/WEBP | 是 |
| file_size | 文件大小（字节） | Integer | >0 | 是 |
| width | 图片宽度（像素） | Integer | >0 | 是 |
| height | 图片高度（像素） | Integer | >0 | 是 |
| add_time | 添加时间 | Timestamp | ISO 8601格式 | 是 |
| use_count | 使用次数 | Integer | ≥0，默认0 | 是 |
| tags | 用户标签列表 | Array<String> | 每个标签最长50字符 | 否 |
| ocr_text | OCR识别的文本内容 | Text | - | 否 |
| semantic_vector | 语义特征向量 | Array<Float> | 固定维度（如512维） | 是 |
| task_id | 任务ID | String(UUID) | 系统自动生成 | 是 |
| task_type | 任务类型 | Enum | 任务类型枚举 | 是 |
| status | 任务状态 | Enum | 任务状态枚举 | 是 |
| total_count | 总数 | Integer | >0 | 是 |
| processed_count | 已处理数 | Integer | ≥0，默认0 | 是 |
| file_list | 文件列表 | Array<String> | 文件路径列表 | 是 |
| error_message | 错误信息 | String | 错误描述信息 | 否 |
| create_time | 创建时间 | Timestamp | ISO 8601格式 | 是 |
| update_time | 更新时间 | Timestamp | ISO 8601格式 | 是 |

## 4.3 术语表
| 术语 | 全称/解释 |
|-----|----------|
| OCR | Optical Character Recognition，光学字符识别，用于提取图片中的文字 |
| 语义检索 | 基于内容理解的检索方式，而非简单的关键词匹配 |
| 特征向量 | 将图片内容转换为数学向量表示，用于计算相似度 |
| Debounce | 防抖，延迟执行机制，避免频繁触发 |
| Toast | 轻量级提示消息，短暂显示后自动消失 |

# 6. 非功能性需求

## 6.1 性能要求
- **响应时间：**
  - 检索响应 < 500ms（图库≤5000张）
  - 检索响应 < 1000ms（图库≤10000张）
  - 界面操作响应 < 100ms
  
- **并发支持：**
  - 本地单用户应用，无并发要求

- **数据量级：**
  - 支持图库规模：10000张以内
  - 索引文件大小：不超过图片总大小的20%
  - 单张图片索引构建：< 2秒

## 6.2 安全要求
- **数据加密：**
  - 索引数据本地存储，无需加密
  - 不涉及用户敏感信息

- **权限控制：**
  - 应用仅访问用户授权的文件夹
  - 遵循操作系统文件权限规则

- **敏感信息处理：**
  - 不收集、不上传任何用户数据
  - 所有数据仅本地存储
  - 性能统计仅用于本地诊断，不进行任何形式的上报
  - 用户可随时查看和删除本地统计数据

- **隐私保护机制：**
  - 应用完全离线运行，无需网络连接
  - 不包含任何用户行为追踪或分析代码
  - 不使用第三方统计服务或SDK
  - 本地统计数据用户可见、可控、可清除

## 6.3 兼容性要求
- **操作系统支持：**
  - macOS 13+（Intel & Apple Silicon）
  - Windows 10+（x64）

- **设备适配：**
  - 屏幕分辨率：≥1280×720
  - 最小窗口尺寸：800×600

- **分辨率支持：**
  - 支持HiDPI/Retina屏幕
  - 自适应缩放

## 6.4 可用性要求
- **系统可用性：**
  - 本地应用，不依赖网络
  - 索引构建失败不影响已有图库

- **容错能力：**
  - 单张图片索引失败不影响其他图片
  - 文件丢失时优雅降级，显示占位图

- **降级方案：**
  - 语义检索失败时降级为标签匹配
  - OCR失败时仍可基于标签和图像特征检索

## 6.5 易用性要求
- **易学性：**
  - 首次使用时显示引导页
  - 主要功能在3次点击内可达

- **易用性：**
  - 核心操作支持快捷键
  - 提示信息清晰明确

- **无障碍访问：**
  - 支持键盘导航
  - 界面元素支持Tab切换

## 7.1 设计规范
- **色彩规范：**
  - 主色：#1890FF（链接、按钮）
  - 成功色：#52C41A

  - 错误色：#F5222D

  - **格式角标色：**
    - GIF：#FF9500（橙色）
    - WEBP：#1890FF（蓝色）

- **字体规范：**
  - 主字体：系统默认（macOS: PingFang SC, Windows: Microsoft YaHei）
  - 标题字号：18px（大）、16px（中）、14px（小）
  - 正文字号：14px
  - 辅助文字：12px
  - **角标字号：10px，加粗**

- **间距规范：**
  - 基础间距单位：8px
  - 常用间距：8px、16px、24px、32px

- **圆角规范：**
  - 卡片：8px
  - 按钮：4px
  - **格式角标：3px**

- **阴影规范：**
  - 卡片：0 2px 8px rgba(0, 0, 0, 0.15)
  - 悬浮：0 4px 16px rgba(0, 0, 0, 0.2)
  - **格式角标：0 1px 4px rgba(0, 0, 0, 0.2)**

## 7.2 交互规范
- **错误提示规范：**
  - 表单错误：输入框下方红色文字提示
  - 操作错误：Toast提示，3秒后自动消失
  - 严重错误：模态框提示，需用户确认

- **成功提示规范：**
  - 轻量操作：Toast提示"已完成"，1.5秒消失
  - 重要操作：Toast提示具体内容，如"已添加5张图片"

- **加载状态规范：**
  - 短时操作（<100ms）：无加载提示
  - 中时操作（100ms~3s）：Loading图标
  - 长时操作（>3s）：进度条
  - **GIF加载：显示首帧灰度图 + Loading图标覆盖**

- **空状态规范：**
  - 显示占位图标和引导文案
  - 提供主要操作入口（如"添加图片"按钮）

- **动画规范：**
  - **GIF播放：**
    - 自动播放延迟：200ms（等待页面稳定）
    - 播放/暂停过渡：淡入淡出100ms
  - 页面切换：淡入淡出200ms
  - 悬浮效果：150ms ease-out
  - Toast提示：滑入200ms，滑出150ms

## 7.3 响应式设计

  - 最小宽度：800px，网格布局调整为2列
  - 标准宽度：1000px~1400px，网格3列
  - 大屏幕：>1400px，网格4列

- **缩略图尺寸适配：**
  - 最小窗口：120×120px
  - 标准窗口：150×150px
  - 大屏幕：180×180px

- **格式角标尺寸：**
  - 固定尺寸：32px × 18px
  - 位置：缩略图右上角，距边缘4px

# 8. 约束与假设

## 8.1 技术约束
- 必须使用本地语义模型，不依赖在线API
- 索引数据必须本地存储，不上传云端
- 支持的图片格式限定为JPG、PNG、GIF、WEBP

## 8.2 业务约束
- 个人用户免费使用
- 不涉及商业表情包版权问题（用户自行承担）
- 遵循各操作系统的应用发布规范

## 8.3 假设条件
- 假设1：用户图库规模在10000张以内
- 假设2：用户设备具备基本的图形处理能力
- 假设3：用户对表情包有一定的记忆（知道大致内容）
- 假设4：用户主要使用中文或英文描述

# 1. 产品概述

## 1.1 背景与问题
在日常聊天场景中，用户往往知道自己想发送哪张表情包，却难以在本地图库中快速定位。对大多数用户而言，收集表情包的方式是“随手保存”，不会主动改文件名、整理文件夹或系统性打标签。当图库规模达到数百上千张时，单纯依赖文件名和时间查找的方式会迅速失效。

## 1.2 产品定义
一款面向随手收藏型用户的本地表情包找图工具。用户无需维护文件名或标签，只需凭记忆输入图上的字、角色名、动作、表情或场景描述，系统即可在本地图库中快速找回目标图片。

## 1.3 目标用户
| 用户类型 | 用户特征 | 核心诉求 | 使用场景 |
|---------|---------|---------|---------|
| 随手收藏型用户 | 经常保存表情包，但几乎不改文件名、不打标签 | 不整理也能找回图片 | 日常聊天、群聊斗图 |
| 重度表情包用户 | 本地存有 500+ 张表情包，日均使用 10 次以上 | 快速定位、快速复制、少量高价值图片可做私有增强 | 日常聊天、社群互动 |
| 特殊对象维护用户 | 会为少量对象做额外维护，如老板、同事、冷门角色 | 角色或对象可稳定召回 | 工作群、固定圈层聊天、冷门角色检索 |

## 1.4 产品目标
**用户目标：**
- 降低用户查找表情包的时间成本
- 在无文件名维护、无标签维护前提下仍可完成基础找图
- 对少量冷门角色或私有对象提供稳定的增强检索能力

**成功指标：**
- 检索响应时间 < 500ms（图库≤5000张）
- 用户找到目标图片的平均操作次数 ≤ 2 次
- 无标签前提下，用户仍可通过台词、角色、动作、场景完成主要搜索任务

## 1.5 范围说明
### 本期做什么
- 本地记忆式搜索功能
- 图库基础管理（添加、删除、查看）
- 标记增强能力
- 私有角色库增强能力
- 快捷复制功能
- 基础设置项

### 本期不做什么
- 不要求用户维护文件名
- 不要求用户批量维护标签库
- 通用知识库驱动的全局搜索中心能力
- 云端同步功能
- 表情包社区/分享功能
- 在线表情包下载
- 移动端适配
- 图片编辑功能

# 2. 系统概要

## 2.1 系统组成
系统由以下核心模块组成：

1. **检索引擎模块**：负责 OCR、图像语义和私有角色增强下的图片匹配
2. **图库管理模块**：负责图片的增删改查
3. **索引构建模块**：负责 OCR、向量和弱提示词索引生成
4. **标记增强模块**：负责手动标记和自动提示词管理
5. **私有角色库模块**：负责冷门角色或私有对象的名称、别名和示例图维护
6. **设置模块**：负责基础配置管理
7. **UI 交互模块**：负责用户界面展示与交互

## 2.2 数据流图
```text
用户输入查询
    ↓
检索引擎分析查询意图
    ↓
选择当前主路：
    - OCR 主路
    - 语义主路
    - 私有角色主路
    ↓
融合辅路补充与轻量热度修正
    ↓
返回结果列表 → UI 展示
    ↓
用户操作（复制）
    ↓
更新使用频次 → 索引数据库

添加图片流程：
用户选择图片 → 图库管理模块 → 索引构建模块
                                    ↓
                            提取 OCR / 生成向量 / 生成自动提示词
                                    ↓
                            写入索引 → 索引数据库
```

# 3. 角色与权限

## 3.1 角色定义
本产品为本地单机应用，不涉及复杂多角色系统。

| 角色名称 | 角色描述 | 主要权限 |
|---------|---------|---------|
| 用户 | 应用使用者 | 所有找图、管理和基础标记能力 |
| 高级用户 / 维护人 | 需要维护私有角色库的用户 | 可维护少量私有角色卡片 |

# 4. 数据字典与术语表

## 4.1 核心业务对象
| 对象名称 | 定义 | 关键属性 | 生命周期说明 |
|---------|-----|---------|-------------|
| 表情包图片 | 用户添加到图库的图片文件 | 图片 ID、文件路径、添加时间、使用次数、OCR 文本、向量索引 | 从用户添加到图库开始，直到用户删除为止 |
| 标记 | 用户或系统为图片添加的辅助检索线索 | 标记文本、来源、是否手动、强弱级别 | 可被用户修改、删除或忽略 |
| 私有角色 | 用户或维护人为冷门角色、私有对象建立的角色卡片 | 名称、别名、示例图、备注 | 角色库记录长期存在，可持续补充和调整 |
| 索引记录 | 图片的 OCR、语义特征向量及相关元数据 | 图片 ID、OCR 文本、特征向量、自动提示词 | 与图片生命周期一致，图片删除时同步删除 |

## 4.2 数据字典
| 字段名 | 字段说明 | 数据类型 | 取值范围/规则 | 是否必填 |
|-------|---------|---------|--------------|---------|
| image_id | 图片唯一标识 | String(UUID) | 系统自动生成 | 是 |
| file_path | 图片文件完整路径 | String | 有效的文件系统路径 | 是 |
| file_name | 原始文件名 | String | 最长 255 字符 | 是 |
| file_format | 图片格式 | Enum | JPG/PNG/GIF/WEBP | 是 |
| file_size | 文件大小（字节） | Integer | >0 | 是 |
| width | 图片宽度（像素） | Integer | >0 | 是 |
| height | 图片高度（像素） | Integer | >0 | 是 |
| add_time | 添加时间 | Timestamp | ISO 8601 格式 | 是 |
| use_count | 使用次数 | Integer | ≥0，默认 0 | 是 |
| marks | 图片标记列表 | Array | 手动标记或自动提示词 | 否 |
| mark_text | 标记文本 | String | 1-50 字符，去除首尾空格 | 是 |
| mark_source | 标记来源 | Enum | manual/auto_hint | 是 |
| ocr_text | OCR 识别的文本内容 | Text | - | 否 |
| semantic_vector | 语义特征向量 | Array<Float> | 固定维度（如 512 维） | 是 |
| private_role_name | 私有角色名称 | String | 1-50 字符 | 否 |
| private_role_aliases | 私有角色别名 | Array<String> | 去重后的别名列表 | 否 |
| private_role_examples | 私有角色示例图 | Array<String> | 图片路径或图片 ID 列表 | 否 |
| private_role_notes | 私有角色备注 | String | 最长 200 字符 | 否 |

## 4.3 术语表
| 术语 | 全称/解释 |
|-----|----------|
| OCR | Optical Character Recognition，光学字符识别，用于提取图片中的文字 |
| 语义检索 | 基于图像向量和文本向量的内容理解检索 |
| 记忆式搜索 | 用户凭台词、角色、动作、场景等记忆线索找图的搜索方式 |
| 私有角色库 | 用于维护冷门角色或私有对象的少量角色卡片集合 |
| 自动提示词 | 系统自动生成的弱辅助检索线索，不等价于最终真值标签 |

# 6. 非功能性需求

## 6.1 性能要求
- **响应时间：**
  - 检索响应 < 500ms（图库≤5000张）
  - 检索响应 < 1000ms（图库≤10000张）
  - 界面操作响应 < 100ms

- **数据量级：**
  - 支持图库规模：10000 张以内
  - 索引文件大小：不超过图片总大小的 20%
  - 单张图片索引构建：< 2 秒

## 6.2 安全要求
- 应用完全离线运行
- 不收集、不上传任何用户数据
- 仅访问用户授权的文件夹

## 6.3 可用性要求
- 单张图片索引失败不影响其他图片
- OCR 失败时仍可基于语义检索找图
- 私有角色缺失时可优雅降级为普通搜索

## 6.4 易用性要求
- 首次使用时显示引导页
- 主要功能在 3 次点击内可达
- 不要求用户理解或调节底层搜索权重

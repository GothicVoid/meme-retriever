# 历史归档说明

本目录用于存放以下文档：

- 已正式生效过、但已被后续口径替代的旧 spec
- 仍有追溯价值、但不应继续出现在当前阅读主路径中的文档

本目录不承担旧编号跳转入口。

当前仓库采用两种历史保留方式：

- `archive/`：保留完整历史正文，供追溯和对比，不再作为当前入口
- `specs/` 根目录兼容入口：保留原编号路径，但正文只剩跳转说明，供旧链接和旧提交继续落地

因此：

- 只有“仍需保留完整历史正文”的旧文档才进入 `archive/`
- 已被收缩成 5 到 10 行跳转说明的编号文档，继续留在根目录，不再移动到 `archive/`
- 若某份旧文档既不需要完整正文保留，也不需要兼容原路径，应直接删除并依赖 git 历史追溯

归档规则：

- 顶部必须标记 `状态：archived` 或 `状态：superseded`
- 必须说明当前应以哪份文档为准
- 不得作为当前开发依据

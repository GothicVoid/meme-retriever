### 5.7.1 高频使用体验与状态反馈功能（优先级：P3）

#### 功能描述
在首页启动态、搜索体验增强和导航重构已完成的基础上，补充一轮面向高频用户的效率增强与关键状态统一反馈。目标不是新增复杂功能中心，而是围绕“导入 -> 搜索 -> 复制 -> 再找回”这条主链路，减少重复操作、降低中断成本，并让用户在加载、失败、恢复等关键场景下始终知道当前发生了什么、下一步能做什么。

本 spec 聚焦会影响产品行为、交互流程和验收口径的 P3 内容，不覆盖纯视觉样式规范。字体、留白、卡片视觉、颜色与层级统一应另行沉淀为设计规范，不在本功能 spec 中展开。

#### 设计目标
- 高频用户可通过键盘和就近入口更快完成“继续搜索、切换结果、复制图片、再次找回”
- 用户复制过的图片和成功搜索过的线索，能够在后续会话中被快速复用
- 图片预览、加载、空状态、错误态、文件丢失和入库中断恢复等关键场景必须提供明确反馈，而不是只留下空白区域
- 用户在主链路中不需要理解内部处理细节，但能判断当前是在加载、失败、等待修复还是可以继续下一步

#### 范围说明
**本期做什么：**
- 增加全局快捷操作，包括快速聚焦搜索、搜索结果键盘导航和快速预览
- 新增最近复制能力，并支持复制成功后的短暂撤销入口
- 强化最近成功搜索记录，用于首页和搜索页继续找回
- 增加主路径中的常用快捷入口，如继续搜索、查看最近复制、查看最近成功搜索
- 统一关键状态反馈，包括加载态、空状态、错误态、文件丢失提示和入库中断恢复提示

**本期不做什么：**
- 不在本期引入复杂命令面板、可自定义快捷键系统或全局工作台
- 不实现基于用户行为的智能推荐排序算法升级，仅做可解释的最近记录与继续使用入口
- 不在本期输出完整视觉设计规范，不规定字体、颜色、阴影等具体样式 token
- 不做跨设备同步的最近复制或搜索记录

#### 主链路说明
**主流程 1：高频用户快速找图并复制**
1. 用户通过快捷键快速聚焦搜索框
2. 用户输入查询，系统展示结果列表
3. 用户可使用键盘在结果间移动焦点，或触发快速预览确认图片
4. 用户执行复制后，系统给出成功提示，并在短时间内提供撤销入口
5. 系统记录最近复制图片与最近成功搜索，供后续继续使用

**主流程 2：用户从首页继续上次使用**
1. 用户重新进入应用或清空当前查询
2. 首页展示最近成功搜索、最近复制和最近使用等模块
3. 用户点击某条记录后直接继续搜索或重新复制目标图片

**主流程 3：用户遇到异常或中断**
1. 用户在搜索、预览、详情、图库管理或入库过程中遇到加载、失败或文件缺失
2. 页面展示对应状态反馈，包括原因说明、下一步建议和可执行入口
3. 若是入库任务中断，系统应提示恢复方式或继续处理入口
4. 用户完成修复后，可返回主链路继续搜索或复制

#### 交互说明
**1. 全局快捷操作**

页面元素：
- 搜索快捷键提示
- 结果列表焦点态
- 快速预览层
- 悬浮快捷入口

交互行为：
- 用户按 `/` 或 `Ctrl+K` 时，系统应将焦点移动到主搜索框，并选中当前输入内容
- 当搜索结果列表存在可见结果时，用户可使用方向键在结果卡之间移动焦点
- 焦点卡应具备清晰的可见态，不要求鼠标悬停
- 用户按 `Enter` 时，默认对当前焦点卡执行复制
- 用户按 `Space` 或 `Shift+Enter` 时，打开当前焦点卡的快速预览
- 用户按 `Esc` 时，优先关闭快速预览；若当前无预览层，则收起相关浮层并保留当前查询
- 悬浮快捷入口应只承接高频动作，如 `继续搜索`、`最近复制`、`最近搜索`，不得演化为第二套导航
- 触屏和鼠标场景下，悬浮快捷入口应不遮挡主内容区域和结果卡关键操作

**2. 最近复制与复制撤销**

页面元素：
- 复制成功 Toast
- 撤销按钮
- 最近复制区块或抽屉

交互行为：
- 用户复制图片后，页面应提示 `已复制`
- Toast 中应在短时间内提供 `撤销` 入口，建议停留 3~5 秒
- 本地复制场景下，`撤销` 的最低要求是撤回本次“最近复制记录”写入，不要求恢复系统剪贴板的旧内容
- 最近复制区应展示最近复制的若干张图片，建议默认 6 条
- 最近复制排序必须按最近一次复制时间倒序
- 点击最近复制图片后，应再次执行复制，并刷新其最近复制顺序
- 若图片文件已丢失，则该记录不应继续出现在最近复制区

**3. 最近成功搜索与继续使用**

页面元素：
- 最近成功搜索区
- 常搜主题区（可选轻量形态）
- 继续使用入口

交互行为：
- 系统应记录真正产生有效结果的搜索词，作为最近成功搜索
- 最近成功搜索与最近搜索历史应区分数据语义：
  - 最近搜索：所有有效非空查询
  - 最近成功搜索：产生可用结果并被用户继续操作过的查询
- 首页应优先展示最近成功搜索，而不是仅展示泛化最近搜索
- 点击最近成功搜索词后，应立即填入并触发搜索
- 若本期增加常搜主题，应使用轻量聚合表达，如按词频选取 3~5 个主题词，不展开复杂推荐逻辑
- 若最近成功搜索为空，则该区块不展示，不使用空壳占位

**4. 图片快速预览**

页面元素：
- 预览弹层
- 大图区域
- 基础元信息区
- 快捷操作区

交互行为：
- 用户可从结果卡、最近复制区、最近使用区等图片入口打开快速预览
- 快速预览应优先用于“放大确认是否是这张”，而不是替代完整详情页
- 预览层至少提供以下操作：复制、查看详情、在文件夹中显示、关闭
- 用户在预览层中切换上下张时，应优先沿用当前来源列表顺序
- GIF 在快速预览层中应正常播放；静态图保持原比例显示
- 预览打开后，背景主页面不应发生误触滚动或焦点跳失

**5. 关键状态反馈统一**

覆盖页面：
- 首页 / 搜索页
- 图库管理页
- 单图详情页
- 导入与入库相关视图
- 快速预览层

状态要求：
- 加载态：必须说明正在处理什么，避免只有旋转图标
- 空状态：必须说明为什么为空，以及下一步建议
- 错误态：必须说明当前动作失败，并提供重试、返回或替代路径
- 文件丢失态：必须明确该图片原文件不存在，并提供重新定位或删除记录入口
- 入库中断恢复态：必须说明有未完成任务，并提供继续处理或放弃清理入口

统一交互规则：
- 状态文案应优先使用任务语言，例如 `正在整理新导入的图片`、`这张图片的原文件找不到了`
- 空状态不能只显示 `暂无数据`，应补充原因和下一步动作
- 错误态不能只显示技术异常字符串，应对外收敛为可理解提示
- 文件丢失态在搜索结果、最近复制、最近使用和详情页中的对外语义必须一致
- 入库中断恢复提示应在用户重新打开应用或回到图库管理页时可见，不能只在任务中断当下闪过一次

#### 页面与状态规则
| 场景 | 触发条件 | 页面行为 | 可执行动作 |
|-----|---------|---------|-----------|
| 快速聚焦搜索 | 用户按 `/` 或 `Ctrl+K` | 聚焦主搜索框并高亮现有输入 | 直接输入、继续搜索 |
| 搜索结果键盘导航 | 结果列表存在可见结果 | 显示结果焦点态，支持方向键移动 | `Enter` 复制、`Space` 预览 |
| 复制成功 | 用户单击图片或在预览中执行复制 | 显示 `已复制` Toast，并短暂展示撤销入口 | 撤销、继续复制 |
| 最近复制展示 | 存在复制记录 | 首页或快捷入口展示最近复制图片 | 再次复制、打开预览 |
| 最近成功搜索展示 | 存在成功搜索记录 | 首页展示最近成功搜索 | 点击重搜 |
| 快速预览 | 用户触发预览 | 打开预览层并锁定背景滚动 | 复制、详情、关闭 |
| 搜索页加载中 | 用户发起搜索且结果未返回 | 展示搜索中状态，不清空结构骨架 | 等待、取消输入 |
| 搜索无结果 | 无可用结果 | 展示失败反馈和下一步建议 | 改搜、去图库管理 |
| 图库为空 | 图库总数为 0 | 展示导入引导空状态 | 导入图片 |
| 原文件丢失 | 图片记录存在但文件不存在 | 展示统一文件丢失态 | 重新定位、删除记录 |
| 入库任务中断 | 上次导入未完成 | 展示恢复提示 | 继续处理、放弃并清理 |

#### 数据要求
| 数据项 | 用途 | 最低要求 |
|-------|------|---------|
| `recent_copies` | 支撑最近复制展示与复用 | 保存图片 ID、最近复制时间、来源页面 |
| `recent_successful_searches` | 支撑首页继续使用 | 保存查询词、最近成功时间、关联点击或复制次数 |
| `preview_source_context` | 支撑快速预览翻页 | 标记当前预览来源列表与当前位置 |
| `index_recovery_state` | 支撑入库中断恢复提示 | 保存未完成任务状态、最近中断时间 |
| `missing_file_state` | 支撑跨页面统一文件丢失反馈 | 至少标记文件是否存在及可恢复动作 |

补充要求：
- `recent_copies` 与现有最近使用数据可共用图片基础信息，但不能只用 `use_count` 代替最近复制顺序
- `recent_successful_searches` 应与 `recent_searches` 分离存储或具备状态字段，避免首页把所有失败查询都当成“继续使用”
- 预览层关闭后，应尽量恢复打开前的焦点位置，便于键盘连续操作

#### 边界情况处理
| 场景描述 | 触发条件 | 系统行为 | 提示信息 |
|---------|---------|---------|---------|
| 搜索框已聚焦 | 用户再次按快捷聚焦键 | 保持焦点，不重复触发动画 | 无 |
| 结果列表为空时按方向键 | 当前没有可导航结果 | 不进入焦点态 | 无 |
| 快速预览来源被刷新 | 用户打开预览后底层结果重新搜索 | 预览保持当前图片，不强制跳图 | `结果已更新`（可选） |
| 复制撤销超时 | Toast 消失后用户再尝试撤销 | 不再执行撤销 | 无 |
| 最近复制记录重复 | 同一图片短时间内被多次复制 | 仅刷新排序时间，不重复新增多条 | 无 |
| 最近成功搜索后续失效 | 该搜索词后续已无结果 | 点击后进入正常失败反馈态 | `这次没有找到结果，换个说法试试` |
| 预览图片文件丢失 | 打开预览时原文件已不存在 | 展示文件丢失态，关闭大图展示 | `这张图片的原文件找不到了` |
| 导入中途退出应用 | 入库任务尚未完成 | 下次启动时展示恢复提示 | `上次有图片还没整理完` |
| 恢复任务失败 | 继续处理时再次报错 | 保留恢复提示并提供放弃清理入口 | `恢复失败，请重试或清理这批记录` |

#### 迭代计划
- 后续可评估是否将最近复制与最近使用做更细致的去重或分组
- 后续可补充轻量命令面板，但前提是不替代现有主导航
- 后续可为常搜主题增加用户手动固定能力，但本期不做
- 后续可将状态反馈样式抽象为统一组件规范，与设计规范文档联动

#### 验收标准
**功能指标：**
- [ ] 用户可通过快捷键快速聚焦主搜索框
- [ ] 搜索结果支持键盘焦点移动，并可通过键盘直接复制或打开快速预览
- [ ] 复制成功后提供短暂可见的撤销入口
- [ ] 首页或快捷入口可展示最近复制记录，并支持再次复制
- [ ] 首页可展示最近成功搜索，并支持一键重搜
- [ ] 图片支持快速预览，且预览层可执行复制、查看详情和关闭
- [ ] 搜索、图库、详情、预览、导入流程中的加载态、空状态、错误态都有明确文案和动作入口
- [ ] 原文件丢失时，跨页面使用统一语义，并提供修复或清理动作
- [ ] 存在未完成入库任务时，用户重新进入应用后能看到恢复提示

**体验指标：**
- [ ] 高频用户可在更少鼠标操作下完成“聚焦搜索 -> 浏览结果 -> 复制图片”
- [ ] 用户重新进入应用后，能快速找回最近复制或最近成功搜索的内容
- [ ] 用户在异常场景下不会面对无解释空白页，且能理解下一步动作

**可靠性指标：**
- [ ] 最近复制顺序按最近复制时间稳定刷新，不因重复操作生成脏数据
- [ ] 最近成功搜索不会混入空白查询或明显失败查询
- [ ] 预览层关闭后，页面焦点和滚动位置可稳定恢复
- [ ] 入库中断恢复提示不会因页面跳转而意外丢失

#### 假设
- 假设复制行为仍以“写入系统剪贴板成功”为准，撤销不要求恢复系统原始剪贴板内容
- 假设最近成功搜索可用“用户执行搜索后有可用结果，且发生点击/复制”等轻量规则定义
- 假设入库中断恢复状态可由本地持久化任务记录提供，不需要单独引入任务中心页面

#### 开放问题
- 快捷聚焦搜索的默认快捷键最终是否同时保留 `/` 与 `Ctrl+K`，还是仅保留一个主快捷键
- 最近复制区块默认放在首页首屏还是通过悬浮入口/抽屉承接，当前建议以不挤压首页主搜索区为原则
- 常搜主题本期是否先只展示静态聚合词，还是延后到有足够真实数据后再开放
